Git Workflow

InspiryThemes

At InspiryThemes a remote git repository always have two branches master and development. master branch represents production code and development branch represents development code.

Every egineer at InspiryThemes is supposed to follow the workflow given below to push changes to development branch. The purpose of this workflow is to avoid conflicts while multiple engineers are pushing changes to same repository.

  • 1) Create a new branch for your changes

    ~$git branch sample-branch

  • 2) Check the list of available branches ( Only if you are not already aware of existing branches )

    ~$git branch

  • 3) Switch to newly created branch

    ~$git checkout sample-branch

  • 4) Make the required code changes and test your code.

  • 5) When you are satisfied with changes, Check the effected files

    ~$git status

  • 6) Stage all the effected files

    ~$git add --all

  • 7) Commit the staged files

    ~$git commit -m "your message"

  • 8) Switch back to development branch

    ~$git checkout development

  • 9) Update development branch from remote repository to have the latest version before merge

    ~$git pull origin development

  • 10) Merge the sample-branch into development branch.

    ~$git merge sample-branch

  • 11) Push the changes in development branch to remote repository

    ~$git push origin development

  • 12) Once done you can delete the sample branch in your local repository

    ~$git branch -d sample-branch
