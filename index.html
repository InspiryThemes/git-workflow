This Git Workflow defines a branching model designed around the project to push changes on the Git Repository. This provides a robust framework for managing larger projects easily while working several people on the same projects.

1. create a new branch for your changes

git branch sample-branch

2. check the list of available branches

git branch

3. switch to newly created branch

git checkout sample-branch

4. make some changes in code

5. check the effected files

git status

6. stage the effected files

git add --all

7. commit the staged files

git commit -m "your message"

8. switch back to development branch

git checkout development

9. update development branch from remote repository

git pull origin development

10. merge the sample-branch into development branch.

git merge sample-branch

11. push the changes in development branch to remote repository

git push origin development

12. once done and tested, delete the sample branch

git branch -d sample-branch
